# Email Redesign Research

## Overview

Three auth flows generate server-side emails to users. This task redesigns the HTML email templates to match the active site theme (including logo) and wires up the actual email sending in the server handlers (previously stubbed with "Phase 5" comments).

## Email-triggering flows

| Page | Route | Handler | Email needed |
|------|--------|---------|-------------|
| `authSignup.jsx` | `POST /signup/assign/group` | `signupAssignGroup` | Welcome email — includes generated password if one was auto-created |
| `authForgotPassword.jsx` | `POST /password/reset` | `passwordReset` | Temporary password email — new random password generated by server |
| `authResetPassword.jsx` | `POST /password/update` | `passwordUpdate` | Password-changed confirmation (fire-and-forget) |

**Note:** `authResetPassword.jsx` requires authentication (redirects to login otherwise) — it is a "change password while logged in" flow.

## Design approach: Option A — client passes `emailTheme`

Emails render server-side but themes live client-side. The auth pages extract a small flat token object (`emailTheme`) from `ThemeContext` and pass it in the POST body. The server uses it to render the HTML without needing to know anything about themes.

### `emailTheme` shape

```js
{
  // Hex color values — defined in auth defaultTheme.js and overridable per site theme
  primaryColor:    string,  // titles, button text   (default: '#1e3a8a')
  accentColor:     string,  // header bg, border, button bg (default: '#dbeafe')
  textColor:       string,  // body copy             (default: '#374151')
  backgroundColor: string,  // outer page bg         (default: '#f4f4f5')

  // Logo
  logoUrl:    string,  // from theme.logo.img (may be '' if unset)
  logoTitle:  string,  // from theme.logo.title || PROJECT_NAME
  siteOrigin: string,  // window.location.origin — used to resolve relative logo URLs
}
```

Theme authors add `auth.emailTheme` with hex values to their theme. Sites without an override inherit the defaults from `packages/dms/src/patterns/auth/defaultTheme.js`.

### Logo resolution

Email clients require absolute image URLs. If `logoUrl` starts with `/`, the server prepends `siteOrigin` before inserting into the `<img src>`. If `logoUrl` is empty, the server renders `logoTitle` as text in the header.

## Auth page visual design (source of truth for defaults)

From `packages/dms/src/patterns/auth/defaultTheme.js`:

- **Card wrapper:** `max-w-lg mx-auto`, `border border-blue-100 shadow-lg rounded-lg`, `p-12`
- **Title:** `font-medium text-blue-900 text-2xl uppercase`
- **Body text:** `font-normal text-blue-500 text-sm`
- **Action button:** `rounded-full bg-blue-100`, label `font-bold text-blue-900 text-xs uppercase`
- **Background:** `rgba(244,244,244,0.96)`

Tailwind color values used as email defaults:

| Token | Hex |
|-------|-----|
| `blue-900` | `#1e3a8a` |
| `blue-100` | `#dbeafe` |
| `gray-700` | `#374151` |
| `gray-100` | `#f4f4f5` |

## Email HTML structure

Table-based inline CSS (required for email client compatibility). No Tailwind.

```
outer <table> — 100% width, backgroundColor, 40px vertical padding
  └─ card <table> — max-width 540px, white, border:1px solid accentColor, border-radius:8px
       ├─ header <td> — accentColor bg, 20px 32px padding
       │    <img> max-height:44px  OR  logoTitle text (primaryColor, bold)
       ├─ body <td>   — white, 28px 32px padding
       │    title     — primaryColor, 18px, uppercase, bold
       │    body HTML — textColor, 14px, line-height 1.6
       │    CTA <a>   — accentColor bg, primaryColor text, rounded-full, 12px uppercase
       └─ footer <td> — #9ca3af, 12px (optional)
```
